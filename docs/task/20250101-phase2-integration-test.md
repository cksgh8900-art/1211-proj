# 작업 완료: Phase 2 통합 테스트 체크리스트

**작업 일시**: 2025-01-01  
**Phase**: Phase 2 - 홈페이지 (`/`) - 관광지 목록  
**작업 항목**: 최종 통합 및 스타일링 - 통합 테스트 체크리스트

## 목적

Phase 2에서 구현된 모든 기능(목록, 필터, 검색, 지도, 페이지네이션)이 정상 작동하는지 확인하기 위한 테스트 체크리스트를 작성합니다.

## 테스트 환경

- **브라우저**: Chrome, Firefox, Safari, Edge (최신 버전)
- **디바이스**: 
  - 모바일 (< 640px)
  - 태블릿 (640px - 1024px)
  - 데스크톱 (≥ 1024px)
- **네트워크**: 일반 모바일 네트워크, WiFi, 오프라인 모드

## 테스트 시나리오

### 1. 기본 목록 조회

- [ ] 페이지 로드 시 기본 관광지 목록이 표시됨
- [ ] 카드 형태로 관광지 정보가 표시됨 (썸네일, 제목, 주소, 타입 뱃지)
- [ ] 로딩 중에는 Skeleton UI가 표시됨
- [ ] 에러 발생 시 에러 메시지와 재시도 버튼이 표시됨

### 2. 지역 필터 적용

- [ ] 지역 드롭다운에서 지역 선택 가능
- [ ] "전체 지역" 선택 시 필터 제거됨
- [ ] 지역 선택 시 해당 지역의 관광지만 표시됨
- [ ] URL에 `area` 파라미터가 추가됨
- [ ] 필터 변경 시 첫 페이지로 리셋됨

### 3. 관광 타입 필터 적용

- [ ] 타입 버튼 클릭 시 선택/해제됨
- [ ] 여러 타입 동시 선택 가능
- [ ] 선택된 타입의 관광지가 표시됨
- [ ] URL에 `type` 파라미터가 추가됨 (다중 선택 시 여러 개)
- [ ] 다중 타입 선택 시 병렬 API 호출로 결과 합쳐짐
- [ ] 필터 변경 시 첫 페이지로 리셋됨

### 4. 정렬 옵션 변경

- [ ] "최신순" 선택 시 수정일 기준 내림차순 정렬
- [ ] "이름순" 선택 시 가나다순 정렬
- [ ] URL에 `sort` 파라미터가 추가됨
- [ ] 정렬 변경 시 즉시 적용됨

### 5. 키워드 검색

- [ ] 검색창에 키워드 입력 가능
- [ ] 엔터 키 또는 검색 버튼으로 검색 실행
- [ ] 검색 결과가 목록으로 표시됨
- [ ] 검색 결과 개수 표시
- [ ] 검색 키워드가 URL에 `keyword` 파라미터로 추가됨
- [ ] 검색 결과 없을 때 안내 메시지 표시

### 6. 검색 + 필터 조합

- [ ] 키워드 + 지역 필터 동시 적용 가능
- [ ] 키워드 + 타입 필터 동시 적용 가능
- [ ] 키워드 + 지역 + 타입 필터 동시 적용 가능
- [ ] 모든 필터가 URL 파라미터로 유지됨
- [ ] 필터 변경 시 검색 키워드 유지됨

### 7. 무한 스크롤

- [ ] 페이지 하단 스크롤 시 자동으로 다음 페이지 로드
- [ ] 로딩 인디케이터 표시
- [ ] 더 이상 항목이 없을 때 "모든 관광지를 불러왔습니다" 메시지 표시
- [ ] 모드 전환 토글로 "무한 스크롤" 선택 가능

### 8. 페이지 번호 방식

- [ ] 모드 전환 토글로 "페이지 번호" 선택 가능
- [ ] 페이지 번호 버튼 클릭 시 해당 페이지로 이동
- [ ] 이전/다음 버튼 동작
- [ ] 현재 페이지가 강조 표시됨
- [ ] 모바일에서 페이지 번호 최적화 (최대 3개)
- [ ] 페이지 정보 표시 (현재 페이지 / 전체 페이지)

### 9. 리스트-지도 연동

- [ ] 데스크톱: 리스트(좌측) + 지도(우측) 분할 레이아웃
- [ ] 리스트 항목 클릭 시 지도가 해당 위치로 이동
- [ ] 리스트 항목 클릭 시 마커가 강조됨
- [ ] 리스트 항목 호버 시 마커가 강조됨
- [ ] 마커 클릭 시 인포윈도우 표시
- [ ] 마커 클릭 시 리스트 항목이 강조됨
- [ ] 관광 타입별 마커 색상 구분

### 10. 모바일 탭 전환

- [ ] 모바일에서 "목록" 탭 클릭 시 목록만 표시
- [ ] 모바일에서 "지도" 탭 클릭 시 지도만 표시
- [ ] 탭 전환이 부드럽게 작동
- [ ] URL에 `view` 파라미터가 추가됨

### 11. 네트워크 에러 처리

- [ ] 오프라인 모드에서 에러 메시지 표시
- [ ] 네트워크 복구 시 재시도 버튼 작동
- [ ] API 에러 시 명확한 에러 메시지 표시
- [ ] 재시도 버튼으로 다시 시도 가능

### 12. 접근성

- [ ] 키보드로 모든 필터 조작 가능
- [ ] Tab 순서가 논리적임
- [ ] Focus 스타일이 명확함
- [ ] 스크린 리더가 모든 요소를 읽을 수 있음
- [ ] ARIA 라벨이 적절히 설정됨

### 13. 반응형 디자인

- [ ] 모바일 (< 640px): 단일 컬럼, 탭 전환
- [ ] 태블릿 (640px - 1024px): 2컬럼 그리드
- [ ] 데스크톱 (≥ 1024px): 분할 레이아웃
- [ ] 모든 화면 크기에서 UI가 적절히 표시됨
- [ ] 터치 영역이 충분함 (최소 44px)
